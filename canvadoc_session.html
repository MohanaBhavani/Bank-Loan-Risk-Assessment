Bank Loan Risk Assessment
Applied Data Intelligence
By
Mohana Bhavani Tatikonda
Project Description
This project helps the bank improve its loan approval process by using machine
learning to assess risk. With a dataset of over 150,000 loan applications and 190+
features, the goal is to predict whether a loan is high or low risk. To do this, the data
was cleaned and prepared before training models like Random Forest, Logistic
Regression, SVM, and XGBoost. These models help identify patterns that indicate
potential defaults, allowing the bank to make better lending decisions, reduce losses,
and manage risk more effectively. By leveraging data-driven insights, the bank can
streamline its approval process and ensure responsible lending. This approach also
enhances customer experience by reducing delays in decision-making. Ultimately, the
project aims to create a more efficient and reliable risk assessment system.

Data Pre-processing and Data Splitting
Data Pre-processing:
 First, I checked the data types and ensured there were no missing values or
empty strings to maintain clean data.
 I then identified and removed outliers using the IQR method to eliminate
extreme values that could affect the analysis.
 To ensure the features are on a similar scale, I standardized them using
StandardScaler, which adjusts the data to have a mean of 0 and a standard
deviation of 1.
Data Splitting:
 I divided the dataset into training and validation sets using an 80-20 split. This
allows us to train the model on one portion of the data and test it on unseen
data to evaluate its performance. I also ensured that the class distribution was
maintained in both sets by using stratification.
Model Description
I chose XGBoost because it's known for its high accuracy and efficiency, especially
with large datasets. It handles complex patterns well and reduces the risk of
overfitting through regularization. XGBoost is great for our project because it
combines speed and performance, often outperforming other models in terms of
predictive power. While SVM and Logistic Regression are good, XGBoost tends to
provide better results for more intricate data.
Model Selection
Resampling Method: We used class weighting in the XGBoost model to handle
imbalanced data. The scale_pos_weight parameter adjusts the weight of the minority
class to balance the model’s focus during training, improving performance on the less
frequent class.
Performance Metrics: To evaluate the model, we used accuracy, precision, recall,
and AUC-ROC score. Accuracy shows the overall correctness, precision indicates the
proportion of correct positive predictions, recall measures the model’s ability to
identify positive cases, and the AUC-ROC score evaluates the model's ability to
distinguish between classes.

classification and compare their performance against previously used models.
K-Nearest Neighbors (KNN): Selected for its simplicity and ability to make non-linear
decisions. It classifies a new observation based on the majority class among its k-nearest
neighbors.
CatBoost: A gradient boosting algorithm optimized for categorical data, robust to
overfitting. It performs well on structured data and efficiently handles missing values
and imbalanced datasets.
Model Selection
The selection of KNN and CatBoost was based on their unique strengths. KNN is
simple and effective for smaller datasets but struggles with high-dimensional data,
whereas CatBoost is optimized for categorical data, offering high accuracy and
efficiency.

Project Deliverable 3
Model Description
LightGBM (Light Gradient Boosting Machine):
LightGBM is a gradient boosting framework that uses decision tree-based models. It is
designed to be highly efficient, offering faster training speeds and lower memory usage
compared to traditional boosting methods. LightGBM uses a histogram-based algorithm
and grows trees leaf-wise rather than level-wise, which allows it to focus on the most
significant splits. This leads to faster convergence and often better predictive accuracy.The
model is capable of handling large datasets, missing values, and categorical features
without the need for extensive preprocessing. These features make LightGBM particularly
effective for structured/tabular data with a mix of feature types.
Model Selection
LightGBM (Light Gradient Boosting Machine): Selected for its speed, scalability, and
high performance on structured data. LightGBM uses a histogram-based, leaf-wise tree
growth algorithm that allows it to converge faster and achieve better accuracy compared to
traditional gradient boosting methods. It handles large datasets, missing values, and
imbalanced classes efficiently, making it well-suited for this classification task.
Resampling Method Utilized During Training
Used Stratified K-Fold Cross-Validation with 5 folds to validate model performance.
This method ensures that each fold maintains the original class distribution, which is
especially important for handling imbalanced target classes.
